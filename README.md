# notification_numbers

Custom-виджет Grist для регистрации извещений об изменении документации на изделия.

## Как это работает

1. Выбираете дату выпуска и изделие (поиск по номенклатуре — таблица
   `Components`/`components`: обозначение и наименование).
2. Ниже подтягивается дерево состава изделия (всё, что входит в него,
   рекурсивно через колонку `primary_use`). У каждой строки — чекбокс,
   сверху — «Выбрать всё».
3. Справа от дерева — кнопки видов документов (СП, СБ, ГЧ, ТУ, РЭ, Д1, ПС,
   ВЭ, ЭТ, ВП, Э5, Э3, Э4, Э2, ИС). Отмечаете компоненты в дереве, включаете
   нужные виды документов и жмёте «Сформировать» — в список справа попадают
   обозначения вида `<decimal_number><код>` (например, `402131.005СБ`).
   Формирование можно повторять несколько раз, список накапливается.
4. Указываете причину изменения и жмёте «Зарегистрировать извещение».

## Поля извещения

| Поле | Тип в Grist | Описание |
|---|---|---|
| Number | Text | Номер извещения вида `ПРЕФИКС.№-ГОД` (например, `ЛМАП.2-2018`) — генерируется автоматически, № — порядковый в рамках префикса и года, без незначимых нулей |
| IssueDate | Date | Дата выпуска извещения |
| Product | Ref (на таблицу номенклатуры) | Изделие, для которого выпущено извещение |
| ProductName | Text (опционально) | Наименование изделия на момент регистрации (денормализовано) |
| DocumentDesignations | Text | Обозначения изменённых документов, через пробел (в реестре отображаются построчно) |
| ChangeReason | Text | Причина изменения (комментарий, редактируется) |
| ChangeNumber | Text | Номер изменения — сквозной счётчик для конкретного изделия |

Названия колонок для извещения сопоставляются через стандартный экран выбора
колонок Grist — физические имена в вашей таблице могут отличаться.

Таблица номенклатуры (`Components`/`components`) подключается напрямую по
имени таблицы (без экрана сопоставления), т.к. из неё строится дерево состава.
Ожидаемые колонки: `name`, `primary_use` (Ref на эту же таблицу, 0 = верхний
уровень), `decimal_number`, `org_prefix`, `serial`. Если у вас другое имя
таблицы — укажите его в настройках виджета (значок ⚙).

## Настройка

1. Добавьте виджет в Grist через **Custom URL**, указав адрес `index.html`
   (например, через GitHub Pages этого репозитория).
2. В экране сопоставления колонок укажите таблицу извещений и колонки для
   каждого поля из таблицы выше.
3. При необходимости откройте настройки виджета (значок ⚙) и укажите точный
   ID таблицы номенклатуры, если он отличается от `Components`/`components`.

## Разработка

Виджет — один файл `index.html` без сборки (vanilla JS + Grist Plugin API).
Открывайте его напрямую как Custom Widget URL в Grist (например, через
`npx serve` локально или GitHub Pages в проде).
